# 2026-04-21 사용자 가이드 해석 설명 보강

## 배경/문제

- 기존 `dist/guide.html`은 탭별 기능 목록은 제공했지만, 사용자가 데이터를 어떻게 읽고 어떤 후속 행동으로 연결해야 하는지에 대한 설명이 부족했다.
- 특히 `② 부킹 트렌드`는 지역별/화주별/영업사원별/AB vs CD 서브탭이 많아, 어느 순서로 봐야 하는지와 지표 간 차이를 명확히 안내할 필요가 있었다.

## 원인/판단

- 대시보드는 영업/운영 의사결정용이므로 단순 화면 설명보다 업무 질문, 지표 의미, 해석 기준, 다음 행동이 함께 있어야 한다.
- 소석률, 3주전BKG/BSA, 실선적률, CM1/TEU, 고수익화주, 구간별고수익, A+B/C+D는 서로 다른 기준의 지표라 혼동 가능성이 높다.

## 결정/계산 로직

- `dist/guide.html`의 개요, 필터, Tab1~5, 데이터 기준 섹션에 다음 관점의 설명을 추가했다.
  - 이 화면이 답하는 질문
  - 지표별 의미와 해석
  - 탭별 권장 사용 순서
  - 이상치 발견 시 다음 확인 화면
  - 혼동하기 쉬운 지표 구분
- `DEVELOPMENT.md`에는 사용자 가이드 작성 기준을 추가했다.
