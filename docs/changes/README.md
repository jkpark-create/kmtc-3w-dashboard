# 개발건별 변경 기록

이 폴더는 `DEVELOPMENT.md`만으로는 길어지기 쉬운 개발 이슈별 맥락을 따로 남기는 공간입니다. 한 개발 건마다 하나의 파일을 만들고, 나중에 NotebookLM 같은 문서 질의 도구에 넣어도 "무엇을 왜 바꿨는지"가 바로 보이도록 작성합니다.

## 작성 원칙

- 파일명은 `YYYY-MM-DD_short-title.md` 형식을 사용합니다.
- 운영 기준으로 확정된 내용은 `DEVELOPMENT.md`에도 반영합니다.
- 사용자에게 보이는 의미, 탭, 문구, 계산 설명이 바뀌면 `dist/guide.html`도 함께 갱신합니다.
- 배포가 끝난 뒤에는 관련 커밋과 검증 결과를 기록합니다.

## 권장 항목

- 배경/문제
- 원인/판단
- 결정/계산 로직
- 변경 파일
- 검증 결과
- 배포/커밋
- 후속 확인사항
