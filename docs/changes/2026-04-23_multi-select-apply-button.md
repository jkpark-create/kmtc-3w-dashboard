# 2026-04-23 멀티셀렉트 적용 버튼 방식

## 배경

- 선적지/선적포트/도착지/도착포트 멀티셀렉트에서 체크박스를 누를 때마다 cascade와 render가 실행되어, 2025년 연간 데이터처럼 큰 데이터셋에서 반응이 느려졌다.
- 사용자가 여러 값을 선택한 뒤 `적용` 버튼으로 한 번만 필터링하는 방식이 필요했다.

## 변경

- `fOri`, `fOriP`, `fDst`, `fDstP` 체크박스 클릭은 `_pending` 값만 갱신하도록 변경했다.
- `적용` 버튼을 추가해 `_pending` 값을 `_selected`로 확정한 뒤 cascade와 render를 한 번만 실행한다.
- `전체` 버튼은 임시 선택만 비우고, `닫기`/외부 클릭/ESC는 임시 선택을 현재 적용값으로 되돌린다.
- 변경 전 상태와 다른 경우만 `적용` 버튼을 활성화하고, 필터 테두리에 dirty 상태를 표시한다.
- 체크박스 기본 동작이 브라우저별로 되돌아가는 것을 막기 위해 `pointerdown`에서 pending 상태를 직접 토글한다.

## 검증

- `dist/index.html` 인라인 스크립트 문법 확인 완료.
- Playwright smoke test로 도착지 `MY` 선택 시 `적용` 전 render 0회, `적용` 후 render 1회와 선택 확정 상태를 확인했다.
